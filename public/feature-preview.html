MatFinder — feature preview

Side-by-side previews of three new features so you can see how they look on real surfaces before triggering them in the live database. Every element follows the design rules: bone outlines only, no arrows, no emojis, JetBrains Mono uppercase for mini-labels, squared corners.

1 · Pin variants on the map

Standard, visited, featured, and favorited pins shown at the same zoom level. The visited pin is the new one — it’s the standard pin plus a thin bone-white ring, signalling “you’ve checked in here.” Subtle by design so it doesn’t compete with featured or favorited.

2 · Weekly check-in badge on gym cards

The new weeklyCheckins prop renders a bone-uppercase “TRAINED THIS WEEK” line next to the location/distance row. Below: a gym card with the new badge, and the same card without it (zero check-ins) for comparison. The badge silently hides when count is zero, so it never adds visual noise to brand-new gyms.

What changed: GymCard now accepts a weeklyCheckins prop (default 0). When > 0 it renders · N TRAINED THIS WEEK in bone uppercase 0.04em. Counts come from /api/checkin-counts — anonymous aggregate of check-ins in the last 7 days. Same badge appears on every card variant (compact portrait, compact-expanded landscape, full desktop) and on the Favorites page. The Popular sort tab now ranks gyms by this same count, tie-broken by Bayesian rating — so Popular ≠ Featured.

3 · Pro vs Standard avatar badge

Tiny tier dot in the bottom-right corner of the avatar trigger. Gold for Pro, mint for Standard, hidden for Free. Bone outline matches the rest of the trigger and keeps it readable on any background. Renders on every page since the trigger lives in the top nav.
