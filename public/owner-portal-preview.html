How it gets built infra

Auth + permissions

  • New Supabase table gym_owners (user_id, gym_id, status, verified_at) — single row per claimed gym.
  • Routes under /owner/[gymId]/* check that the current user has a verified row in gym_owners for that gym; everyone else gets a 404.
  • RLS on gym_overrides: an authenticated user can UPDATE only the row whose gym_id is in their gym_owners set.
  • Featured listing subscription is the gating signal — if their subscriptions.tier is featured AND status = active AND they're verified, the portal opens.

Schedule editing

  • New Supabase table gym_overrides (gym_id, open_mats jsonb, website, phone, instagram, hero_image, updated_at).
  • /api/gyms already merges schedule-overrides.json; we add one more layer that pulls from gym_overrides last so owner edits win.
  • Cache TTL on /api/gyms drops from 1h to ~5 min once owner edits exist, so changes propagate fast.

Analytics

  • Already shipping: the events table is live and ingesting card_open, pin_tap, ig_click, directions_click from production right now.
  • /api/owner/analytics?gym_id=…&range=7d aggregates counts server-side using the service-role key (RLS keeps clients out).
  • Bar chart + heatmap + breakdown panels are simple SQL group by date_trunc('day', occurred_at) queries — no external charting library, all SVG.
  • Check-in heatmap pulls checkins.checked_in_at bucketed by day-of-week × hour-of-day.

Pricing

  • The Featured Listing subscription ($29.99/mo) IS the gym-owner SKU. It includes Schedule Editor + Analytics + Featured pin + Priority placement.
  • Personal Pro ($6.99/mo or $59.99/yr) is unrelated. A gym owner can be on Free or Pro for their own use; Featured is independent.
