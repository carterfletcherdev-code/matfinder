Cross-card consistency rules (locked)

  • Header layout: name on the left (18px / 22px on mobile), rating right-aligned with star + count. Same on every variant.
  • Button hierarchy: ONE big primary action (full-width, 48px). Then a uniform 3-button secondary grid (Directions / Call / Instagram, all 36px equal width). RSVP lives inside the open-mat panel and stays small (32px) so it doesn't compete with check-in.
  • Open-mat panel: always green-accented when present. RSVP button + going avatar stack on the same row. The same component renders on all 3 cards.
  • Status indication: When in session → orange top strip (v2 shows it). When open mat today → green dot in meta row. When closed → muted gray dot. Universal.
  • Wrong info? always bottom-right of the meta footer, ghost text-only style. Phase A treatment. Will move to overflow menu later.
  • Featured tier (v3): single word "Featured" — no "Sponsor". Gold border + gold strip + gold "verified by gym" badge. Otherwise identical to v1.

My take on which to build

Build v1 as the default and v2 as a power-user toggle. v1 is what 80% of users will see. v2's day-strip is a great progressive-disclosure feature for users who want to plan further ahead — could live as an "expanded" mode after they tap the card the first time.

v3 (Featured) is just a CSS variant of v1 — same component, different border + top strip. Easy to flip on/off based on the gym's tier.

m1 (Mobile sheet) is what the card expands into when tapped on mobile. Same component, different layout.
