What changed from v1

  • Name is now a website link. Subtle external-arrow icon next to it makes the affordance clear; hover lifts the icon and brightens the name. Click stops propagation so the rest of the card-click still selects the gym. Falls back to a plain non-link name if the gym has no website.
  • Zero emojis. All buttons use proper Lucide-style stroke SVG icons — directions, phone, instagram, heart, check, flag, pin, external-link. Same icon set works in light + dark mode.
  • Day strip — 2 versions side-by-side. Card A drops it entirely (cleaner). Card B/C keeps it, but it's now functional — clicking a day swaps the open-mat panel for that day's full class list (Card C shows the post-click state).
  • Mobile sheet refined. Added quick stat row (classes/week · open mats · matfinders trained), full street address line (tap to open Maps), and switched action-grid emojis to SVG icons.

My take

Card A (no day strip) wins for the default list view. The day strip is real estate that costs us ~80px and only pays off for the small slice of users planning ahead — most just want "is there an open mat soon?", which the green panel already answers.

Card B/C's day-strip pattern is great for the full /gym/[slug] page where there's room to breathe and the user is already deep into the gym's details. So: A on the list, day-strip on the gym page. Best of both.

Decision needed

  • Lock A as the standard card? (My recommendation.)
  • Lock m1 as the mobile sheet? (Anything else missing?)
  • Move day-strip pattern to the upcoming /gym/[slug] page — fully implemented + filterable?
